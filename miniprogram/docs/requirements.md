# 需求分析报告

## 1. 用户角色
| 角色 | 典型用户 | 核心诉求 |
| :--- | :--- | :--- |
| 学生 Ben | 本科生、研究生 | 降低阅读门槛，比喻化解释，准备汇报 |
| 研究者 Alice | 博士、科研人员 | 深度拆解方法，批判性质询，跨文献关联 |
| 职场 David | 分析师、工程师 | 30分钟速读多篇，提取商业洞察与行动项 |

## 2. 功能需求
详见之前整理的“细化各界面功能”文档，覆盖首页、阅读器、笔记、个人中心四大模块，包括划词解析、笔记锚定、角色切换、多格式导出等。

## 3. 非功能需求
- 划词响应 < 2 秒（骨架屏缓冲）
- 翻页流畅，按需渲染
- 同一文本解释缓存 24h
- 安全合规，数据隔离

## 4. 里程碑
MVP（第1-4周）→ 三模式完善（第5-8周）→ 跨论文关联（第9-12周）
